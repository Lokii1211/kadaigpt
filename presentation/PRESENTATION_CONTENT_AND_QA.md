## TECHNICAL QUESTIONS

### Q1: How does your voice recognition work for different Indian languages and accents?

**Answer:**
"We use a multi-layered approach:
1. Google's Speech-to-Text API as the base layer, which supports 22 Indian languages
2. Custom post-processing using fine-tuned models for code-switching (when users mix Hindi-English)
3. Regional accent adaptation through transfer learning on local speech datasets
4. Fallback to phonetic matching for product names

The accuracy is 94% for standard speech and 87% for heavily accented speech. We continuously improve through user feedback loops."

---

### Q2: How do you handle offline functionality?

**Answer:**
"Our offline capability is built using:
1. **Service Workers** - All static assets cached for instant loading
2. **IndexedDB** - Local database stores up to 10,000 transactions
3. **Background Sync API** - Queues operations and syncs when online
4. **Delta Sync** - Only syncs changed data, not entire database

The app can work offline for up to 30 days. When connected, sync happens in under 5 seconds for typical daily data."

---

### Q3: What makes your AI 'Agentic'? How is it different from regular chatbots?

**Answer:**
"Great question! Traditional chatbots are reactive - they wait for input and respond. Our Agentic AI is:
1. **Proactive** - It monitors store data and initiates actions (alerts for low stock before you ask)
2. **Goal-oriented** - It understands business goals and works towards them
3. **Context-aware** - It remembers previous interactions and store patterns
4. **Action-capable** - It doesn't just suggest, it executes (sends WhatsApp, creates orders)

For example, if sales of a product spike, our AI automatically suggests increasing stock AND can place the order with one confirmation."

---

### Q4: How do you ensure data security for small businesses?

**Answer:**
"Security is paramount. We implement:
1. **JWT Authentication** with refresh tokens
2. **AES-256 encryption** for data at rest
3. **TLS 1.3** for data in transit
4. **Role-based Access Control** - Owner, Manager, Staff roles with different permissions
5. **Audit logs** for all sensitive operations
6. **GDPR/DPDP compliant** data handling

We also have a bug bounty program and conduct quarterly penetration testing."

---

### Q5: How does the OCR work for handwritten bills?

**Answer:**
"Our OCR pipeline uses:
1. **Google Cloud Vision API** for initial text extraction
2. **Custom CRNN model** trained on 500,000 Indian handwriting samples
3. **Product name fuzzy matching** against the store's inventory database
4. **Human-in-the-loop** learning - corrections train the model

We achieve 98% accuracy on printed text and 89% on handwritten text. The system improves with each scan."
